# Community Acquisition Playbook

Last updated: 2026-07-15

RaidBench acquisition should not depend on spam, hidden promotion, mass replies, or private-message scraping. The workable path is: find real player questions, answer them well, track the pain point, and only add a RaidBench link when the community rules and context allow it.

## Core Channel Strategy

| Channel | Role | Link policy | Recommended use |
| --- | --- | --- | --- |
| Google / Bing SEO | Main long-term inbound channel | Fully allowed on our own site | Build free pages around repeated problems, then convert with credit packs. |
| Reddit | Demand discovery and selective helpful replies | Link only when rules and context allow it | Answer first, disclose relationship, link only as optional reference. |
| Steam Discussions | Demand discovery, not promotion | Avoid direct promotion links | Read pain points; reply with short neutral advice only if useful. |
| Discord servers | Community listening and permitted help | Only in allowed channels or with moderator permission | Join relevant communities, use self-promo/help channels if they exist, never DM strangers. |
| YouTube comments | Demand discovery and creator partnerships | Avoid repetitive comment links | Look for repeated questions under guide videos; contact creators only with a clear collaboration offer. |
| Paid ads | Later validation channel | Allowed through ad platform policies | Start only after organic pages show clicks and conversion intent. |

## Compliance Rules

1. Never mass-post the same message.
2. Never use fake accounts, vote manipulation, self-bots, or paid upvotes.
3. Never hide that RaidBench is our site if linking to it.
4. Never DM strangers with promotional links.
5. Never post links in communities that ban self-promotion.
6. Never ask users to bypass game rules, exploit bugs, buy accounts, buy in-game items, or use cheating tools.
7. For Steam Discussions, treat the platform as research-first because Steam broadly prohibits advertising/commercial activity in community behavior.
8. For Discord, do not use bots or automated accounts for outreach; each message should be human-reviewed.

## Reply Ladder

Use this ladder for every community interaction.

| Level | When to use | What to post | Link allowed |
| --- | --- | --- | --- |
| L0 Research only | The thread bans self-promo or the answer is sensitive | Do not reply; save the question and summarize the pain point. | No |
| L1 Pure answer | User asks a clear gameplay question | Give direct steps in plain English. | No |
| L2 Answer plus disclosure | Link is useful and rules allow it | Answer first, then disclose: "I maintain RaidBench, so treat this as my own checklist." | Optional |
| L3 Moderator-approved post | A subreddit/server has a resource channel or permission was granted | Post a full guide/resource with disclosure. | Yes |
| L4 Paid collaboration | Creator, newsletter, or community owner accepts sponsorship | Use agreed wording and tracking link. | Yes |

## English Reply Workflow For A Chinese Operator

The owner does not need to write English manually. Use this workflow:

1. Capture the player message, thread URL, game, patch/version context, and any community rules.
2. Translate the user intent into Chinese.
3. Classify the request:
   - Gameplay question
   - Build/base audit
   - Resource route or calculator need
   - Patch/update confusion
   - Payment/support question
   - Not safe to answer
4. Draft an English reply with one of the templates below.
5. Check whether a link is allowed.
6. Post manually from the owner account, or have Codex operate the already-open browser only after the owner confirms the exact reply.
7. Record result: posted, removed, ignored, upvoted, replied, converted, or banned by rules.

## Reply Templates

### Pure Helpful Answer

Hi, the fastest way to debug this is to separate the problem into three checks:

1. What changed after the last patch or server reset?
2. Which part is failing first: damage, survivability, resource supply, or routing?
3. Can you repeat the issue twice under the same setup?

For your case, I would start with [specific recommendation]. If that does not fix it, check [second recommendation].

Short answer: [direct answer].

I would not change everything at once. Test only one variable first:

1. [step]
2. [step]
3. [step]

If the game was patched recently, re-check this against the current version notes before rebuilding around it.

### Moderator Permission Request

Hi mods, I maintain a small free guide/calculator site called RaidBench.

I found several repeated questions in this community around [topic]. I would like to share a free checklist that answers the question without selling in-game items, boosting, accounts, or currency.

Would it be acceptable to reply with the link when it directly answers a thread, with a clear disclosure that I maintain the site? If not, I will keep replies text-only.

## Daily Operating Rhythm

Morning:

- Check 10 to 20 fresh questions across selected communities.
- Save only high-intent questions.
- Add 3 to 5 new intent cards.

Afternoon:

- Draft replies.
- Post only L1/L2 replies that are clearly allowed.
- Convert repeated questions into new free SEO pages.

Evening:

- Review which replies got responses.
- Update content status: Current, Needs Review, Deprecated, or Archived.
- Add one improvement to a free page or calculator.

## What Codex Can Do

Codex can:

- Translate English player questions into Chinese.
- Summarize customer intent and urgency.
- Draft English replies.
- Check a community rule page before suggesting a link.
- Convert repeated questions into new RaidBench pages.
- Operate an already-open browser with owner approval for a specific reply.
- Maintain the intent log and content backlog.

Codex should not:

- Auto-post bulk comments.
- Create fake accounts.
- Evade moderation systems.
- Send unsolicited DMs.
- Hide commercial interest.
- Promise official game support or guaranteed outcomes.

## Rule References

- Reddit Rules: participate authentically, follow community rules, and do not spam or manipulate content.
- Reddit self-promotion guidance: do not start by submitting your own links; keep own-content links as a minority of participation.
- Steam Online Conduct: commercial activity and advertisements are prohibited behavior.
- Discord Community Guidelines: unsolicited bulk messages, self-bots, and inauthentic engagement are prohibited.
